Login • Connectivity

API returning HTTP 429
API • Rate limits
1. Respect the Retry-After header before retrying.
2. Implement client‑side rate limiting per API key.
3. In the dashboard, open Settings → API → Usage to review your current limit tier.
Desktop app stuck on “Syncing workspace”
Desktop • Sync
1. From the tray icon, choose Help → Run diagnostics.
2. If you see “queue backlog”, clear the local cache from Help → Advanced → Reset local state.
3. Re‑sign in and confirm the workspace appears in the selector.
Rotate production API keys safely
Security • Keys
1. Create a new key in Settings → Security → API keys.
2. Deploy it alongside the old key and monitor logs.
3. Once traffic is stable, revoke the old key and archive the rotation record.

Troubleshooting guides Step‑by‑step fixes for common problems.
“Connection lost” appears every few minutes
▼
What it means: the client is losing its websocket connection to our edge nodes.

Quick checks:
• Confirm your clock is in sync with an NTP server—large skews can invalidate TLS sessions.
• If you’re on Wi‑Fi, test on a wired connection to rule out local instability.

Advanced:
1. Run the built‑in network test: Help → Diagnostics → Network.
2. If you see packet loss > 2%, capture a 60‑second trace and attach it to your ticket.
3. Ask your network team to allow persistent connections to wss://edge.novatech.cloud.

App won’t start after an update
▼
Symptoms: the app window never appears, or closes immediately after launch.

Steps:
1. On Windows, open Event Viewer → Windows Logs → Application and look for recent NovaTech errors.
2. If you see SideBySide or missing runtime errors, reinstall using the “Full bundle” installer.
3. On macOS, run: open /Applications/NovaTech.app --args --safe-mode to start without extensions.
4. If safe mode works, disable third‑party extensions one by one until you find the culprit.

Webhook deliveries are delayed or missing
▼
Checklist:
1. Confirm your endpoint returns a 2xx status within 10 seconds.
2. Check the Webhook logs page for recent failures and replay attempts.
3. Ensure your TLS certificate is valid and not using deprecated ciphers.

Debugging tip: temporarily point the webhook to a request inspector (such as a generic HTTPS echo endpoint) to confirm events are being sent as expected.

Single sign‑on (SSO) fails for some users
▼
Common causes:
• The user is not assigned to the NovaTech app in your IdP.
• The email domain in the SAML assertion doesn’t match the verified domains in your workspace.

Steps:
1. In your IdP, confirm the user is assigned and the NameID is set to their primary email.
2. In NovaTech, open Settings → Security → SSO and run the configuration validator.
3. If you see “Domain mismatch”, add the domain to your verified list or update the IdP mapping.
